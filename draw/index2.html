  • ✕ 절대 손대지 않음
해당 파일 라이브러리화된 파일 · 최근 PMS · 최근 탭/Web 구조

B
애매한 파일 — 최소 정리
  • ✓ 파일명 앞에 접두어 붙이기
  • ✓ 페이지명만 정리 (WIP / FINAL / ARCHIVE)
  • ✕ 레이어 안 건드림
  • ✕ 내용 수정 없음
판단 질문 "지금 쓰고 있나, 아니면 언젠가 쓸 것 같아서 두고 있나?"
후자라면 C로 강등
C
옛날 파일 — 보관 중심
  • ✓ Archive 폴더로 이동
  • ✓ 파일명에 ARCHIVE_ 붙이기
  • ✕ 열어서 고치지 않음 — 절대
핵심 관점 과거 파일은 기록이지, 작품이 아니다.
완성도를 높이려는 충동을 무시한다.

Section 04

정리 철학

원칙 01
살아있는 것만 관리한다
과거 파일을 현재 기준으로 고치려 하지 않는다. 살아있는 파일 중심으로 좋은 시스템을 만드는 것이 목표다.
원칙 02
파일을 열면 고치고 싶어진다
C등급 파일은 절대 열지 않는다. 보관은 보관이다. 분류 세션에서는 홈 화면만 본다.
원칙 03
완벽한 정리보다 지속 가능한 루틴
30분 안에 끝낸다. 타이머가 울리면 멈춘다. 다음 주에 이어서 한다.
원칙 04
A등급이 기준이다
최근의 잘 정리된 파일이 앞으로의 기준점. 새 파일은 처음부터 이 기준으로 만든다. 과거 파일을 이 기준으로 올리려 하지 않는다.
원칙 05
언제 멈출지 미리 정해둔다
정리가 너무 잘 되면 계속 하고 싶어진다. 시간 제한이 곧 완료 기준이다. "다 못 했다"가 아니라 "오늘 몫을 했다".
This system is for 예전 것도 기준 맞춰야지 ✕
